Human Resource Management System

Industry-standard, scalable, and secure web application built with Django and Bootstrap for managing end-to-end workforce operations.

Introduction

The HRMS (Human Resource Management System) is a comprehensive enterprise application designed to streamline HR operations. It centralizes employee records, tracks attendance and leaves, manages organizational holidays, and automates monthly payroll processing.

The system is built upon a robust, layered Object-Oriented Programming (OOP) architecture, ensuring clean separation of concerns, high security, and maintainability for future scaling.

Core Modules

Employee Management

Maintain comprehensive profiles for Full-Time, Part-Time, and Contract employees. Track departments, designations, joining dates, and employment status seamlessly.

Leave Management

Allow employees to apply for various leave types (Annual, Sick, Casual). Enforces overlap validation, maintains status tracking (Pending, Approved, Rejected), and auto-calculates remaining balances.

Holiday Management

Organize organizational calendars with National, Regional, and Company holidays. Quickly view upcoming time-off across the company.

Payroll Processing

Automate monthly salary calculations. Generates structured payslips detailing Basic Pay, HRA, Allowances, and automated deductions (PF, TDS). Tracks payment status and histories.

Industry-Standard Architecture

The system departs from standard procedural Django setups, utilizing a highly structured, scalable Service-Oriented Architecture (SOA). This prevents "Fat Models" and "Fat Views", ensuring business logic is isolated, testable, and reusable.

The Service Layer Paradigm

Django Views are strictly limited to handling HTTP requests and formatting responses. All data manipulation, validation rules, and specialized calculations live inside Service Classes. This ensures operations are atomic and insulated from presentation logic.

# Example: Payroll Processing located strictly in services.py

class PayrollService:
    @staticmethod
    @transaction.atomic
    def create_payroll(validated_data: dict) -> Payroll:
        # 1. Enforce business rules (no duplicates)
        PayrollService._check_duplicate(employee.pk, month, year)

        # 2. Automated Calculations via internal utilities
        payroll = Payroll(**validated_data)
        payroll.gross_salary = calculate_gross_salary(payroll)
        payroll.net_salary = calculate_net_salary(payroll)

        # 3. Commit securely
        payroll.full_clean()
        payroll.save()
        return payroll

Security & Core Infrastructure

The system uses a dedicated core application to house centralized utilities, enforcing strict security boundaries and preventing monolithic code buildup.

Security Hardening

Configured against common web attacks. Includes forced HTTPOnly Session & CSRF cookies, strict X-Frame-Options (Clickjacking prevention), and MIME-type sniffing protections.

Query Optimization

Heavy usage of Custom Managers overriding default QuerySets. Implements automated select_related and prefetch_related chaining to prevent N+1 query performance bottlenecks on scale.

Custom Domain Exceptions

Instead of generic database errors, the core routing utilizes granular error handling like LeaveConflictError, InvalidPayrollStateError, and DuplicateEmployeeIDError.

Reusable View Mixins

DRY implementation via HRMSLoginMixin, HRMSCreateMixin, and context injectors. Every view automatically processes success messages and ensures authorization.
